# Secure Workflow Verification for Embedded Systems

This repository contains the implementation of a **secure workflow verification architecture** for embedded systems, developed as part of a Master's thesis.

The goal of the project is to ensure that a program executed on a microcontroller follows a **valid execution workflow**, preventing tampering with sensor data and program logic.

The approach combines:

- encrypted variables
- secure API execution
- execution trace generation
- workflow verification against a control graph

---

# Project Idea

The system records the sequence of executed operations (APIs) together with the variables involved in each operation.

The execution trace is then compared with a **workflow graph extracted from the original C program**.  
If the trace is compatible with the graph, the execution is considered valid.

Otherwise the execution is rejected.

---

# Architecture

The system is composed of three main components.

## 1 Device Runtime (Embedded side)

The embedded device executes the program using a **secure runtime layer**.

Modules currently implemented:

device/
crypto.c → AES encryption layer
int_var.c → encrypted variable abstraction
trace.c → secure execution trace generation


These modules guarantee that:

- variables are always encrypted
- operations are logged
- execution traces cannot be forged

---

## 2 Graph Extractor

The workflow graph is extracted automatically from the C program using static analysis tools.

The graph represents:

- variables (places)
- API operations (transitions)

This representation is similar to a **Petri Net model**.

---

## 3 Execution Verifier

The verifier receives:

- the workflow graph
- the execution trace generated by the device

It then checks whether the trace is **derivable from the graph**.

Output:

OK → execution valid
NO → execution tampered


---

# Security Goals

The system protects against:

- modification of sensor values
- manipulation of program variables
- alteration of the execution workflow
- replay or injection of fake operations
